## Mudancas aplicadas

### App

- A autorizacao admin agora depende apenas de `app_metadata.role = 'admin'` ou `app_metadata.roles` contendo `admin`.
- O cliente privilegiado do Supabase foi centralizado em helper dedicado.
- URLs de retorno do checkout deixaram de depender de `Origin`/headers dinamicos e agora usam `NEXT_PUBLIC_SITE_URL`.
- Uploads de imagem ganharam validacao compartilhada de MIME e tamanho antes de irem ao bucket.
- O webhook Stripe passou a reconstruir os itens de forma confiavel a partir dos line items da sessao, com idempotencia por `stripe_session_id`.

### Banco

- A conta `admin@admin.com` recebeu bootstrap em `raw_app_meta_data` com `role = admin`.
- Todas as policies administrativas relevantes foram recriadas com base em `app_metadata`, nunca `user_metadata`.
- `orders` e `order_items` ficaram com leitura do proprio usuario e escrita reservada a `service_role` ou admin legitimo.
- O bucket `product-images` manteve leitura publica, mas escrita/atualizacao/remocao ficaram restritas a admin.
- `decrement_stock` agora usa `SET search_path = public`, opera em `stock_quantity`, retorna `boolean` e so permite `EXECUTE` para `service_role` e `postgres`.

## Evidencia ao vivo

### Auth

- Usuario bootstrapado: `admin@admin.com`.
- `raw_app_meta_data` atual: `{"role":"admin","roles":["admin"],"provider":"email","providers":["email"]}`.

### Policies

- `products`, `product_variations`, `product_images`, `store_settings` e `categories`:
  - leitura publica preservada onde esperado;
  - `ALL` restrito a authenticated com claim admin via `app_metadata`.
- `orders`:
  - `SELECT` do proprio usuario com `auth.uid() = user_id`;
  - `ALL` apenas para admin.
- `order_items`:
  - `SELECT` por subconsulta dos pedidos do proprio usuario;
  - `ALL` apenas para admin.

### Storage

- Bucket `product-images`:
  - `public = true`
  - `file_size_limit = 5242880`
  - `allowed_mime_types = image/jpeg, image/png, image/webp, image/avif, image/gif`
- `storage.objects`:
  - `SELECT` publico apenas para `bucket_id = 'product-images'`
  - `INSERT`, `UPDATE` e `DELETE` apenas para admin autenticado

### Funcao critica

- `public.decrement_stock`:
  - `security_type = DEFINER`
  - `search_path` controlado no SQL versionado
  - `EXECUTE` concedido apenas para `service_role` e `postgres`
