## Correcoes aplicadas

### App

- `generateMetadata()` da raiz agora usa `NEXT_PUBLIC_SITE_URL` como base canonica/OG.
- Categorias e produtos passaram a sinalizar `notFound()` ja na fase de metadata, reduzindo o risco de fallback tardio em rotas inexistentes.
- `billing.disputed` agora preserva `status = disputed` no pedido.
- A conta do cliente e o admin passaram a reconhecer `disputed` como status explicito.
- A pagina de sucesso deixou de tratar qualquer pedido existente como automaticamente confirmado.
- `middleware.ts` foi migrado para `proxy.ts` para compatibilidade com Next 16.

### Banco

- Nova migration de hardening aplicada para:
  - criar helper `public.is_admin()`;
  - reescrever policies de RLS com `(select auth.uid())` / `(select public.is_admin())`;
  - remover policies SELECT administrativas redundantes;
  - adicionar indices de FK em `orders`, `order_items` e `payment_attempts`;
  - fixar `SET search_path = public` nas funcoes do sync Abacate;
  - restringir `EXECUTE` das funcoes internas do sync Abacate;
  - alinhar storage policies do bucket `product-images` ao helper admin.
